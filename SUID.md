SUID (Set User ID)
**Что делает:** при запуске программы она получает права владельца файла
Пример - chmod 4755 /path/prog
Классика: `/usr/bin/passwd` (владелец root, чтобы обычный юзер мог менять пароль).

## `2xxx` — SGID (Set Group ID)

**Что делает:**
на файле (программе): при запуске получает **группу владельца файла** (effective GID)

на каталоге все новые файлы/папки внутри **наследуют группу каталога** (удобно для общей папки команды)

## `1xxx` — Sticky bit

**Что делает (на каталоге):** удалять/переименовывать файлы внутри могут только:  
владелец файла, владелец каталога или root.  
Даже если у других есть "W" на каталог.


### Быстрая шпаргалка

- `chmod 4755 file` → SUID (`s` у владельца)
    
- `chmod 2755 file/dir` → SGID (`s` у группы)
    
- `chmod 1777 dir` → sticky (`t` у остальных)
