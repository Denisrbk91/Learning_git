Показать, какие процессы слушают порты -
TCP
sudo ss -ltnp

UDP
sudo ss -lunp

Найти, кто слушает конкретный порт
ss -ltnp 'sport = :8080' или  ss -lunp 'sport = :8080'

Найти процесс по PID 
ps -fp PID


`-l` — **listening**: показать только **слушающие** сокеты (серверные порты).
`-t` — **TCP** сокеты.
`-u` — **UDP** сокеты.
`-n` — **numeric**: не пытаться резолвить имена (хосты/сервисы), показывать **числа** (например `80`, а не `http`). Быстрее и понятнее.
`-p` — **process**: показать **процесс**(ы), которые держат сокет (PID/имя).

Так же можно использовать 
sudo lsof -i :8000
sudo netstat -lntup| grep LISTEN

Проверить на удаленном хосте порты:
Linux 
nc -vz server.example.com 22
nc -vz server.example.com 443

Test-NetConnection server.example.com -Port 443
Test-NetConnection server.example.com -Port 22
