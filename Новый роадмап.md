## Принципы обучения (чтобы реально прокачаться)

1. **Каждая тема = мини-проект + чеклист** в `infra-lab/README.md`.
    
2. **Каждую неделю**: 1) изучил, 2) сделал лабу, 3) сломал, 4) починил, 5) записал “как чинить”.
    
3. Учишься не “по роликам”, а через тренажёры и реальные задачи.
    

---

## Фаза 0 — База стенда (1 вечер)

**Цель:** уметь подключаться по SSH-ключу, понимать что где лежит, иметь репозиторий-лабу.

**Практика:**

- SSH сервер в WSL, ключи, `authorized_keys`, права.
    
- README: “как подключиться / как перезапустить ssh / как чинить сеть”.

## Фаза 1 — Linux как рабочий инструмент (1–2 недели)

**Цель:** уверенно жить в терминале и чинить базовые поломки.

### Что выучить (минимум)

- файловая система, права (`chmod/chown`), пользователи/группы
    
- процессы (`ps/top/kill`), systemd (`systemctl`)
    
- логи (`journalctl`, `/var/log`)
    
- сеть: IP/DNS/маршруты, `ss`, `curl`, `dig/nslookup`
    

### Тренажёры (обязательно)

- **Linux Upskill Challenge** — 20 уроков именно под “админить сервер из консоли” [GitHub+1](https://github.com/livialima/linuxupskillchallenge?utm_source=chatgpt.com)
    
- **SadServers** — реальные “сломанные сервера”, которые надо диагностировать и чинить (это прям DevOps-стиль) [SadServers+1](https://sadservers.com/?utm_source=chatgpt.com)
    
- **OverTheWire (Bandit)** — прокачивает уверенность в Linux/SSH/файлах/командах в игровом формате [OverTheWire](https://overthewire.org/wargames/?utm_source=chatgpt.com)
    

### Мини-проекты в `infra-lab/`

- `01-linux-basics.md`: команды, права, systemd, логи.
    
- `02-network-cheatsheet.md`: как проверить DNS/порт/маршрут.
    
- “Сломай и почини”: намеренно убей сервис, почини и опиши шаги.

## Фаза 2 — Git + скрипты (1 неделя)

**Цель:** автоматизировать рутину и вести инфраструктуру как код.

### Практика

- Git: ветки, PR/MR, конфликты
    
- Bash: переменные, условия, циклы, пайпы
    
- (опционально, но полезно) Python для мелких утилит
    

### Задачи

- Напиши скрипт: “проверить доступность сайта, порт 22/80, DNS; вывести отчёт”.
    
- Скрипт бэкапа папки + логирование + ротация.

## Фаза 3 — Docker (1–2 недели)

**Цель:** уметь упаковывать приложение и поднимать стек локально.

### Практика

- Dockerfile, образ/контейнер, volumes, networks
    
- docker-compose: 2–3 сервиса вместе
    

### Тренажёры

- **Play with Docker** и их classroom-лабы (прям в браузере) [Docker+1](https://www.docker.com/play-with-docker/?utm_source=chatgpt.com)
    
- Подборка **dockerlabs** с hands-on лабораторками [dockerlabs](https://dockerlabs.collabnix.com/?utm_source=chatgpt.com)
    

### Мини-проект

- `docker-compose` со стеком: `nginx + app + postgres` (или `nginx + prometheus + grafana` позже).
    
- README: как поднять/остановить/посмотреть логи/починить сеть контейнера.

## Фаза 4 — CI/CD (1 неделя)

**Цель:** собрать pipeline, который билдит, тестит и собирает Docker-образ.

### Практика

- GitHub Actions или GitLab CI (для портфолио GitHub Actions проще начать)
    
- Артефакты, переменные, секреты
    

### Тренажёр/дока

- Официальная дока **GitHub Actions** (workflows, jobs, triggers) [GitHub Docs+1](https://docs.github.com/en/actions?utm_source=chatgpt.com)
    

### Мини-проект

- Pipeline: на push → линтер → build Docker image → (опционально) push в registry (можно локально/ghcr).
    
- README: как запустить workflow и где смотреть логи.

## Фаза 5 — IaC и конфигурации (2 недели)

**Цель:** “сервер поднимается из кода”, а не руками.

### 5A) Ansible (быстро даёт результат)

- Playbook: установить nginx, открыть порт, положить конфиг, перезапустить сервис
    

### 5B) Terraform (особенно если хочешь в облака)

- state, providers, variables, modules
    
- (для практики можно начать с локальных примеров/учебных провайдеров, а потом — облако)
    

### Тренажёры

- Terraform tutorials (HashiCorp Developer) [HashiCorp Developer+1](https://developer.hashicorp.com/terraform/tutorials?utm_source=chatgpt.com)
    
- Подготовка по Terraform Associate тоже даёт структурированный набор тем [HashiCorp Developer](https://developer.hashicorp.com/terraform/tutorials/certification-003/associate-study-003?utm_source=chatgpt.com)
    

### Мини-проект

- `ansible/`: роль “base-server” (users, ssh hardening, packages, fail2ban опционально)
    
- `terraform/`: “поднять VM + сеть” (если есть доступ к облаку/песочнице)

### Практика

- Метрики: Prometheus
    
- Дашборды: Grafana
    
- Логи: хотя бы базовые подходы (сбор/фильтрация)
    

### Мини-проект

- В docker-compose подними Prometheus+Grafana и помониторь:
    
    - сам хост/WSL (если удобно)
        
    - контейнеры
        
- README: “как понять, что сервис умер” + “как алертить”.

## Фаза 7 — Kubernetes (2–4 недели, параллельно можно с остальным)

**Цель:** уметь деплоить приложение в кластер, понимать базовые сущности.

### Практика

- Pods, Deployments, Services, Ingress, ConfigMap/Secret
    
- basic troubleshooting: `kubectl describe`, events, logs
    

### Тренажёры

- **Killercoda** (интерактивные окружения/сценарии в браузере, много по k8s) [Killercoda](https://killercoda.com/killer-shell?utm_source=chatgpt.com)
    
- **Killer Shell** (симуляторы экзаменов уровня CKA/CKAD — очень жёсткая, но крутая практика) [Killer Shell](https://killer.sh/?utm_source=chatgpt.com)
    

### Мини-проект

- Задеплой приложение из Docker-фазы в k8s.
    
- Сделай обновление (rolling update), откат, проверка readiness/liveness.

## Что я бы предложил тебе прямо сейчас (следующий шаг)

Раз ты уже поднял SSH и ведёшь `infra-lab/`, давай сделаем **первый реальный артефакт**:

**`infra-lab/README.md`**

- как подключаться по SSH
    
- как проверить, что sshd жив (`systemctl status ssh`, `ss -tlnp | grep :22`)
    
- как чинить “не пускает по ключу” (права на `.ssh/authorized_keys`)
    
- как чинить “пропал интернет в WSL” (`wsl --shutdown` в PowerShell и т.п.)
    

Если скинешь, что у тебя сейчас за структура `infra-lab/` и какой стек хочешь для капстоуна (nginx+app+postgres или что-то другое), я распишу **помесячный/понедельный план** под твой темп (например, “1–1.5 часа в будни + 1 длинная сессия в выходные”) и дам точные задания “сделай → проверь → сломай → почини”.
