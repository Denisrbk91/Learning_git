# Чек-лист готовности: Linux инженер технической поддержки (РФ)
*Версия: 2025-12-31*

Этот документ — самопроверка по навыкам Linux для техподдержки: **Junior (L1)**, **Middle (L2)** и **Senior/Expert (L3)**, а также список типовых практических задач для собеседований.

---

## Как пользоваться
- Каждый пункт = **1 балл** (умею сделать руками + объяснить «почему»).
- Оценка:
  - **Junior (12 пунктов):** 9–12 — уверенно; 6–8 — стажировка/джун; <6 — добрать базу
  - **Middle (14 пунктов):** 11–14 — уверенно; 8–10 — «джун+»; <8 — добрать практику
  - **Senior (16 пунктов):** 12–16 — уверенно; 9–11 — крепкий middle; <9 — подтянуть глубину

---

# 1) Junior Linux Support (L1 / новичок)

## ОС и файловая система
- [ ] Навигация и поиск: `ls`, `find`, `grep`, `less`, `tail -f`
- [ ] Права/владельцы/маски: `chmod`, `chown`, `umask`, понимание `rwx`
- [ ] Архивы и работа с логами: `tar`, `gzip`, `xz`, поиск по логам

## Пользователи
- [ ] Пользователи/группы: `useradd/usermod`, `groups`, `id`, что такое `sudoers`
- [ ] Сброс пароля/разблокировка учётки, различаю локальную учётку и доменную

## Сервисы и логи
- [ ] `systemctl status/start/stop/enable`, понимаю что такое unit
- [ ] Логи: `journalctl -u <svc>`, знаю где искать логи приложений, умею понять «почему не стартует»

## Пакеты и обновления
- [ ] Установка/удаление/обновление пакетов (`apt`/`dnf`/`yum`), репозитории на базовом уровне
- [ ] Понимаю отличие `deb` и `rpm` (концептуально)

## Сеть (база)
- [ ] Диагностика: `ip a`, `ip r`, `ping`, `traceroute`, `nslookup/dig`, `ss -lntp`
- [ ] Понимаю **DNS/DHCP/NTP**: что делают и как проверить (например `timedatectl`/`chronyc`)

## Базовый Bash
- [ ] Могу написать простой скрипт: собрать логи/проверить сервис/сделать отчёт (условия, циклы, переменные)

---

# 2) Middle Linux Support (L2 / опытный инженер)

## Диагностика (глубже)
- [ ] Умею фильтровать `journalctl` по времени/сервису/уровню, понимаю уровни логирования
- [ ] Процессы/ресурсы: `top/htop`, `ps`, `nice/renice`, `free`, `vmstat`, `iostat` (базово)
- [ ] Диск: `df -h`, `du -sh`, `lsof` («кто держит файл»), понимаю inode-проблемы

## Сеть (уже серьёзно)
- [ ] `tcpdump` (снять трафик, фильтр по порту/хосту)
- [ ] TLS/сертификаты: проверка цепочки/сроков/ошибок (минимум `openssl s_client`)
- [ ] Firewall: где смотреть правила (iptables/nft), как понять что «режет»

## Веб/БД/наблюдаемость (дежурный набор)
- [ ] Nginx/Apache: по логам и конфигу найти причину 502/504/403/SSL проблем
- [ ] PostgreSQL (минимум): доступность, базовые логи/коннекты, причины «слишком много соединений»

## Пакеты/зависимости/репозитории
- [ ] Нахожу причины «пакет не ставится»: конфликты, зависимости, репы, ключи
- [ ] Понимаю где искать конфиги/файлы пакета, postinst, service unit

## Интеграции
- [ ] LDAP/AD на уровне поддержки: bind/baseDN, проверка `ldapsearch`, диагностика подключения
- [ ] Kerberos/SSO (концептуально): время, DNS, realm/SPN — умею собрать факты и проверить очевидное

## Процесс поддержки
- [ ] Пишу хороший тикет/эскалацию: шаги, факты, логи, «expected vs actual», приоритет
- [ ] Делаю авто-сборщики диагностики (bash/python), веду KB/FAQ

# 3) Senior / Expert Support (L3)

## Глубокая отладка
- [ ] `strace` (куда «стучится», что не открывается, диагностика syscall)
- [ ] Core dump/`coredumpctl`, базово читать backtrace (символы/модули/контекст)
- [ ] Производительность: CPU steal/IO wait/saturation/memory pressure/OOM и диагностика

## Архитектура и надёжность
- [ ] Понимаю HA/балансировку/healthchecks на уровне поддержки продукта
- [ ] Провожу RCA: таймлайн → первопричина → корректирующие меры
- [ ] Предлагаю профилактику повторов: алерты, лимиты, настройки, гайды

## Сеть (уровень «на равных»)
- [ ] TCP глубже: retransmits, MTU/MSS, keepalive, NAT/прокси-цепочки
- [ ] Интерпретирую capture, отличаю DNS/route/firewall/app issue

## Контейнеры/платформы
- [ ] Docker: сети/volumes/лимиты/логирование; отличаю проблему контейнера от host
- [ ] Kubernetes (если актуально): logs/events, probes, service/ingress (базово)

## Продуктовая поддержка
- [ ] Воспроизвожу баг и пишу баг-репорт для dev (min repro, версии, условия, логи)
- [ ] Могу читать чужой код/конфиги для поиска причины (не обязательно писать фичи)
- [ ] Стандартизирую «диагпакеты»: что собирать, как маскировать секреты

## Коммуникация и лидерство
- [ ] Веду сложные кейсы end-to-end, координирую несколько команд
- [ ] Улучшаю документацию/онбординг/туллинг, обучаю младших
- [ ] SLA/приоритизация/эскалации — уверенно

# 4) Практические задачи для собеседования (тренажёр)

## A) Linux core (быстрое расследование)
- [ ] «Сервис не стартует» → `systemctl status`, `journalctl -u`, конфиг/права/порты
- [ ] «Диск кончается» → `df -h`, `du -sh /*`, `lsof | grep deleted`, ротация логов
- [ ] «Высокая нагрузка» → `top`, `ps`, `free`, `vmstat`, поиск виновника
- [ ] «Нет прав на файл» → `ls -l`, `id`, группы, ACL/`sudo` (если применимо)
- [ ] «Файл занят» → `lsof`, `fuser`, оценка безопасного рестарта

## B) Сеть
- [ ] «Не резолвится домен» → `dig/nslookup`, resolver, кеш, доступность DNS
- [ ] «Порт недоступен» → `ss -lntp`, firewall, route, `curl -v`, `nc`
- [ ] «Таймауты» → `traceroute`, MTU, `tcpdump`
- [ ] «Время съехало, Kerberos ломается» → `timedatectl`, `chronyc`, reachability NTP
- [ ] «SSL ошибка» → `openssl s_client`, SNI, цепочка, сроки, hostname

## C) Веб/приложения
- [ ] «502/504 на Nginx» → upstream/таймауты/логи Nginx+приложения+сокеты
- [ ] «403/401» → права/SELinux/AppArmor (если есть), auth-слой, заголовки
- [ ] «Логи растут гигабайтами» → ротация, уровень логирования, лимиты

## D) Базы данных (часто встречается)
- [ ] «Postgres не пускает» → max connections, pg_hba.conf, auth, логи
- [ ] «Медленные запросы» (support-уровень) → собрать факты: plan/locks/connections

## E) Автоматизация и процесс
- [ ] Скрипт «собери диагностику» (версии, `systemctl`, логи за 30 мин, сеть, диск)
- [ ] Эскалация: что приложишь, какие гипотезы, что уже исключил
- [ ] KB-статья: симптомы → причины → проверка → решение → профилактика

## F) AD/LDAP (если корпоративка)
- [ ] «Пользователь не логинится в домен» → DNS/время/bind/политики/блокировки
- [ ] «LDAP интеграция отвалилась» → сертификаты/LDAPS, firewall, baseDN, creds, trace
